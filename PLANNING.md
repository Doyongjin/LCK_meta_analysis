# LCK Victory Formula & Meta Analysis Tool — 기획서

## 1. 프로젝트 개요

### 핵심 질문
"누가 이겼나"가 아닌 **"어떤 메타적 선택이 승리를 이끌었나"**

### 목표
LCK 경기 데이터를 활용해 패치 버전별 팀 성향, 밴픽 전략, 진영 및 픽 순서에 따른 승률 상관관계를 분석하고 시각화한다.
1
### 사용 목적
- 팬 입장의 흥미로운 분석 콘텐츠
- 코치/분석가의 전략 수립 참고
- 데이터 분석 포트폴리오

---

## 2. 분석 레이어

```
레이어 1 — 메타 레벨
  패치별 강챔, Blue/Red 승률 추이, 밴픽 트렌드

레이어 2 — 팀 레벨
  팀별 진영 선호도, 밴픽 성향, 15분 지표, 오브젝트 우선순위

레이어 3 — 선수 레벨
  선수별 챔피언 풀, 밴 내성, 스페셜리스트 챔피언
```

## 3. 핵심 지표

| 지표 | 설명 |
|------|------|
| Gold@15 | 15분 기준 골드 차이 |
| CSD@15 | 15분 기준 CS 차이 |
| XPD@15 | 15분 기준 경험치 차이 |
| Pick/Ban Presence | 픽 + 밴 합산 등장률 |
| Side Win Rate | 블루/레드 진영별 승률 |
| First Pick Efficiency | 선픽 vs 후픽 승률 |
| Object Control | 첫 드래곤/전령 장악률 |

## 4. 분석 시나리오 (A~H)

### A — 핵심 챔피언 밴 시 승률 영향
선수 주력 챔피언이 밴됐을 때 팀 승률과 15분 지표가 얼마나 달라지는지 측정.
- 주력 챔피언 밴된 경기 vs 안 밴된 경기 승률 비교
- 15분 골드 하락 → 승률 하락 연결 분석

### B — 진영별 챔피언 선택 변화
같은 선수가 블루/레드 진영에서 고르는 챔피언이 어떻게 달라지는지 분석.
- 블루 전용 / 레드 전용 / 공통 챔피언 목록화

### C — 패치 적응 속도
패치 출시 후 팀별로 강해진 챔피언을 얼마나 빠르게 픽에 반영하는지 측정.
- 패치 출시 → 첫 채택까지 경기 수 계산
- PBI 기준 강챔 선정

### D — 저격 밴 패턴
특정 팀을 상대할 때 상대방이 밴 1~3순위에 우리 팀 선수 주력 챔피언을 집중하는 패턴 분석.

### E — 패치별 승리 공식
이긴 팀들의 공통 조건(진영, 픽 순서, 15분 지표, 오브젝트)을 수치화해서 패치별 승리 공식 역추적.
- Logistic Regression으로 각 요소의 기여도 산출

### F — 선수별 밴 내성 지수 (라인별 기준 분리)
챔프폭, 주력 의존도, 밴 시 승률 하락폭을 종합해 0~100 점수 산출.
- 라인별 LCK 평균 기준선 따로 계산 (미드 기준과 원딜 기준이 다름)
- 서포트: 유틸/탱커/인게이저 역할군 내 비교
- 탑: 스플릿형/합류형 성향 구분 후 비교

### G — 팀 색깔 × 승률 상관관계
F 결과를 팀 단위로 합산해 팀 색깔 분류 후 진영/패치/상대팀 조합별 승률 교차 분석.

| 색깔 유형 | 특징 |
|---------|------|
| 캐리 의존형 | 특정 선수 챔프폭 좁고 밴 시 승률 급락 |
| 시스템형 | 5명 모두 챔프폭 넓고 밴 내성 강함 |
| 초반 압박형 | 15분 지표 항상 앞서고 첫 오브젝트 장악률 높음 |
| 후반 역전형 | 15분 지표 평범하지만 후반 승률 높음 |

### H — 스페셜리스트 챔피언 판별
단순히 많이 쓰는 챔피언이 아닌, LCK 동일 라인 평균 대비 이 선수가 유독 잘하는 챔피언 식별.
- 기준: 최소 플레이 횟수 충족 + 초과 승률 + 15분 지표 우위
- 연계: A(밴 타격도), D(저격 밴 패턴), F(밴 취약도)

## 5. 데이터 소스

| 소스 | 용도 | 수집 방식 |
|------|------|----------|
| Oracle's Elixir | 경기 전체 데이터 | AWS S3 자동 다운로드 (공식 경로 검증 필수) |
| Leaguepedia | pick_order 보완 + 패치 변경 내역 | 위키 API 자동 수집 |
| Community Dragon | 챔피언 아이콘 + 패치 날짜 | GitHub 자동 수집 |

> **주의:** Oracle's Elixir는 2026년 4월 DNS 해킹 이력 있음. 다운로드 시 공식 AWS S3 경로(amazonaws.com) 반드시 확인.
> **미확인:** Oracle's Elixir 2026 CSV에 pick_order 컬럼 존재 여부 → 수집 단계에서 직접 확인 필요. 없을 경우 Leaguepedia로 보완.

### 데이터 기간 및 전략
```
2024~2025 데이터 → 선수/팀 역사 프로파일 (맥락 레이어)
  - 선수 중심 프로파일: 이적과 무관하게 개인 성향 추적
  - 감쇠 가중치: 2024(0.3) / 2025(0.6) / 2026(1.0)

2026 데이터 → 현재 분석 대상
```

## 6. 규정 변경 처리

### First Selection (2026년~)
블루 진영과 선픽이 분리됨. 패배팀이 진영 또는 픽 순서 중 하나를 먼저 선택.
- `rule_era`: `pre_2026` | `first_selection`
- `first_selection_choice`: `side` | `pick_order` | `null`(2025 이전)
- 진영 단독 효과 vs 픽 순서 단독 효과 분리 분석 가능

### 피어리스 밴픽
시리즈 내에서 한 번 픽된 챔피언은 나머지 경기에서 양팀 모두 사용 불가.
- 2024 LCK: 일반 밴픽
- 2025 LCK Cup (2025.01~): 하드 피어리스 최초 도입
- 2025 정규시즌 이후~2026 현재: 피어리스 전면 적용
- `draft_type`: `standard` | `fearless`

## 7. DB 스키마

### 그룹 1 — 기준 데이터
```
seasons         시즌/스플릿 정보 (rule_era 포함)
patch_versions  패치 버전 + 출시일
champions       챔피언 기본 정보
teams           팀 정보
players         선수 정보 (팀과 무관하게 존재)
```

### 그룹 2 — 이력 데이터
```
player_team_history   선수-팀-시즌 소속 이력 (이적 추적 핵심 테이블)
  player_id / team_id / season_id / role / joined_date / left_date
```

### 그룹 3 — 경기 데이터
```
series          시리즈 (BO3/BO5, draft_type 포함)
games           단일 경기
game_teams      팀 단위 경기 기록 (경기당 2행)
  side / pick_order / first_selection_choice / result / gold_at_15 / first_dragon / first_herald
game_participants  선수 단위 경기 기록 (경기당 10행)
  player_id / champion_id / position / gold_at_15 / cs_diff_at_15 / xp_diff_at_15
picks_bans      밴픽 순서 전체 (경기당 최대 20행)
  phase / global_order / team_pick_order / side / champion_id
```

> **픽 순서 재구성:** 드래프트 순서는 고정 매핑표로 처리.
> 2026 이후는 game_teams.pick_order 확인 후 두 가지 매핑 중 선택 적용.

### 그룹 4 — 분석 파생 데이터
```
champion_meta   패치별 PBI 지수 (pick_rate / ban_rate / win_rate / pbi_score)
player_profiles 선수 성향 프로파일 (2024~2025 기반 누적)
```

## 8. 분석 모듈

각 시나리오별 계산 방식:

| 시나리오 | 계산 방식 | 결과물 |
|---------|----------|-------|
| A | 밴된 경기 vs 안 밴된 경기 승률 비교 | 선수별 밴 타격도 |
| B | 블루/레드 경기 분리 후 챔피언 목록 비교 | 진영별 챔피언 성향 맵 |
| C | 패치 출시 후 팀별 첫 채택까지 경기 수 | 팀별 메타 적응 속도 지수 |
| D | 상대팀 밴 1~3순위 vs 우리 팀 주력 챔피언 겹침 | 저격 밴 대상/빈도 |
| E | Logistic Regression (진영+픽순서+지표 → 승패) | 패치별 승리 공식 |
| F | 챔프폭 + 주력 의존도 + 밴 취약도 합산 (라인별 기준) | 선수별 밴 내성 점수 0~100 |
| G | F 결과 팀 합산 + 15분 지표 + 오브젝트 패턴 | 팀 색깔 프로파일 |
| H | LCK 동라인 평균 대비 초과 승률 + 지표 우위 | 선수별 스페셜리스트 챔피언 |

## 9. 시각화 기획

### 시나리오별 차트
| 시나리오 | 차트 형태 |
|---------|----------|
| A | 챔피언 아이콘 + 밴 전후 승률 막대 |
| B | 블루/레드 탭 전환 챔피언 목록 (겹치는 챔피언 색 표시) |
| C | 패치 출시일 기준 팀별 채택 시점 타임라인 |
| D | 상대팀 × 우리 팀 선수 저격 밴 격자 히트맵 |
| E | 요소별 기여도 막대 (패치 선택 시 변화) |
| F | 선수별 거미줄 그래프 (라인별 탭, LCK 평균선 포함) |
| G | 2D 좌표에 팀 배치 (초반강도 × 밴내성) |
| H | 챔피언 버블 차트 (크기=플레이 횟수, 높이=초과 승률) |

### 추가 화면

**매치 상세**
- 밴픽 타임라인 (글로벌 순서)
- 블루/레드 진영 + 선픽/후픽 표시
- 15분 지표 비교
- 분석 태그 자동 표시 (저격 밴 감지 / 스페셜리스트 등장 / 초반 압박 패턴 등)

**주간 리뷰**
- 주차별 순위 변동 꺾은선 그래프
- 이번 주 메타 트렌드 (픽률/밴률 급등 챔피언)
- 이번 주 주목할 밴픽 패턴
- 라인별 지표 1위 선수

**픽 제안 / 약점 표시**
- 1단계 (현재): 과거 패턴 기반 — 이 팀이 이 상황에서 과거에 고른 챔피언 TOP 3
- 2단계 (고도화): 밴픽 흐름 + 상대 구성 + 메타 조합 추천
- 약점 챔피언: 상대가 이 챔피언 들고 왔을 때 우리 팀 승률 기반 자동 표시

## 10. 프로젝트 구조

```
1단계 — 기능 검증
  FastAPI (백엔드) + Streamlit (프론트)
  분석 기능 하나씩 만들며 검증

2단계 — 완성본
  FastAPI 유지 + React + D3.js로 프론트 교체
  (백엔드 변경 없이 프론트만 교체)
```

### 구현 순서
```
1. PostgreSQL DB 구축 (스키마 생성)
2. 데이터 수집 ETL
   ├─ Oracle's Elixir CSV 자동 다운로드 (공식 경로 검증)
   ├─ Leaguepedia 위키 API 수집
   └─ Community Dragon 챔피언/패치 데이터 수집
3. 분석 모듈 개발 (A~H 순서대로)
4. Streamlit 화면 연결 (1단계)
5. React + D3.js 프론트 고도화 (2단계)
```
