# LCK Victory Formula — 모델 한계 및 신뢰도 가이드

분석 결과를 해석하기 전에 반드시 읽어야 할 제약 사항 목록입니다.

---

## 1. 데이터 소스 한계

### Oracle's Elixir (주 데이터)
- **픽 순서(`pick_order`) 컬럼**: 2026년 CSV에 존재 여부 미확인. 없을 경우 선픽/후픽 분석 정확도 하락.
- **데이터 갱신 주기**: 최신 경기가 수집된 시점 이후 경기는 반영되지 않음. ETL 재실행 전까지 누락.
- **DNS 보안 주의**: 2026년 해킹 이력 있음. 반드시 공식 S3 주소(`amazonaws.com`)에서만 다운로드.
- **지역 필터**: LCK 데이터만 분석하며, LCK 참가 팀의 국제전(MSI, Worlds) 경기는 포함되지 않음.

### Leaguepedia (보조 데이터)
- **PatchNotes 테이블**: API 구조 보장 안 됨. 실패 시 로컬 캐시(`patch_changes_cache.json`)로 폴백하며, 캐시가 오래됐을 경우 부정확.
- **픽 순서 보완**: Oracle's Elixir에 픽 순서 없을 때 Leaguepedia에서 보완하지만, 두 소스 매핑 오류 가능.

### Community Dragon (챔피언 아이콘 / 패치 날짜)
- **패치 출시일**: `known_patches` 딕셔너리에 수동 하드코딩됨. 새 패치 추가 시 코드 수동 업데이트 필요. 자동화되지 않음.
- **내부 버전 ID**: 시스템 내부는 `16.08` 형식 사용 (Oracle's Elixir 기준). 화면 표시는 `26.8` (Riot 공개 명칭). 두 형식이 혼용될 수 있음.
- **챔피언 아이콘**: URL이 Community Dragon CDN에 의존하며, CDN 변경 시 이미지 깨짐.

## 2. 시나리오별 한계

### A. 밴 시 승률 영향
| 항목 | 내용 |
|------|------|
| 분석 기준 | 선수의 전체 기간 TOP N 챔피언 기준 (기본 N=3) |
| 밴 정의 | "상대팀이 밴한 경우"만 계산. 아군 밴으로 못 쓴 경우는 제외 |
| 샘플 부족 | 밴된 경기가 0~2경기인 경우 승률 신뢰도 극히 낮음 |
| 인과관계 | 밴 → 승률 하락의 인과관계가 아닌 상관관계만 측정. 상대가 강할 때 밴을 더 많이 할 수 있음 |

### B. 진영별 챔피언 성향
| 항목 | 내용 |
|------|------|
| 진영 불균형 | 블루/레드 게임 수가 불균형할 경우 비교 왜곡 |
| 전략적 맥락 무시 | 특정 패치에서 강한 챔피언을 진영과 무관하게 픽하는 현상 미반영 |
| 샘플 부족 | 특정 챔피언을 한 진영에서만 1~2회 플레이한 경우 의미 없음 |

### C. 패치 적응 속도 (PBI)
| 항목 | 내용 |
|------|------|
| PBI 공식 | `(win_rate - 0.5) × pick_rate / (1 - ban_rate)` |
| 평균 승률 가정 | 0.5(50%)로 고정. 실제 LCK에서 진영에 따라 편향될 수 있음 |
| 분석 대상 | 패치별 TOP 5 챔피언만 분석. 소수 챔피언의 메타 변화 감지 불가 |
| 패치 초기 불안정 | 패치 첫 주 경기 수가 적어 PBI 수치가 극단적으로 나타날 수 있음 |
| 적응 속도 정의 | "첫 픽까지 경기 수"로 측정. 팀 스케줄, 휴식주 등의 영향을 배제하지 않음 |

### D. 저격 밴 패턴
| 항목 | 내용 |
|------|------|
| Early ban 기준 | 글로벌 순서 1~6번(1페이즈)으로 고정. 2페이즈 밴 전략 감지 불가 |
| 주력 챔피언 기준 | 선수의 TOP 3 챔피언만 추적 |
| 의도 판단 불가 | 상대가 해당 챔피언을 "저격"으로 밴했는지, 메타 픽이어서 밴했는지 구분 불가 |
| 피어리스 영향 | 피어리스 드래프트에서는 동일 챔피언 재픽 불가 → 자연스럽게 밴 패턴 변형 |

### E. 패치별 승리 공식 (Logistic Regression)
| 항목 | 내용 |
|------|------|
| 최소 경기 수 | **20경기** (임의 설정). 통계적 유의미한 결론을 위해서는 최소 100경기 이상 권장 |
| 모델 단순성 | Logistic Regression은 비선형 관계 포착 불가 |
| 다중공선성 | `gold_at_15`, `first_dragon`, `first_herald`, `first_tower`는 서로 상관관계가 높아 계수 해석 왜곡 가능 |
| 결과물 | 계수(coefficient)는 방향성 참고용. 절대값으로 해석 불가 |
| 인과관계 없음 | "15분 골드 우위가 승리를 만든다"가 아니라, "승리하는 팀이 15분 골드 우위인 경향"이 있는 것 |

### F. 밴 내성 지수 (0~100점)
| 항목 | 내용 |
|------|------|
| 가중치 | 챔프폭 0.25 / 주력 의존도 0.28 / 밴 시 승률 하락 0.27 / 골드 안정성 0.20 — **근거 없는 임의값** |
| 라인별 기준값 | player_profiles 테이블 데이터 없으면 하드코딩된 기본값 사용 (2024~2025 메타 기반, 2026 부정확 가능) |
| `avg × 2` 배수 | 비교 기준을 "평균의 2배"로 설정한 것은 임의적. 극단값 선수에게 불리하게 작동 |
| 밴 데이터 없음 | 밴된 경기가 0이면 승률 하락 = 0으로 처리 (유리한 방향으로 편향) |
| 점수 50점 기본값 | 데이터 없는 선수는 50점 반환. 실제 능력과 무관 |

### G. 팀 색깔 분류
| 항목 | 내용 |
|------|------|
| 분류 기준 | `ban_resistance < 40` → 캐리 의존형, `≥ 65` → 시스템형, `gold > 500 AND obj > 0.55` → 초반 압박형, 나머지 → 후반 역전형 |
| 임계값 근거 없음 | 40, 65, 500, 0.55 모두 경험적 추정값. 데이터 기반 최적화 미실시 |
| 4가지로만 분류 | 실제 팀 전략은 훨씬 복잡하며, 상황에 따라 다른 전략을 구사하는 팀도 하나의 색깔로만 분류 |
| 팀 구성 변동 | 이적 후에도 이전 시즌 데이터가 색깔 계산에 포함될 수 있음 |
| 순환 의존성 | G의 색깔 분류가 F(밴 내성) 점수에 의존하므로, F의 오차가 G에 그대로 전파됨 |

### H. 스페셜리스트 챔피언
| 항목 | 내용 |
|------|------|
| 최소 경기 수 | **3경기** — 통계적 신뢰도 35% 미만. 사실상 의미 없는 샘플 크기 |
| 점수 배분 | 초과 승률 50점 / 골드 우위 30점 / 안정성 20점 — **근거 없는 임의 배분** |
| LCK 평균 없는 챔피언 | 비교 기준 wr = 0.5, gold = 0으로 처리 → 희귀 챔피언에 유리하게 편향 |
| 포함 기준 | 세 지표 중 하나라도 양수면 스페셜리스트로 포함 → 과도하게 많이 포함될 수 있음 |
| 시즌 간 비교 | 선수가 여러 시즌 걸쳐 쌓은 데이터 혼합 시 현재 실력을 왜곡 |

## 3. 공통 통계적 한계

### 샘플 크기
- LCK는 연간 약 200~300경기 규모. 시나리오별 필터링(특정 패치, 특정 팀, 특정 챔피언) 후 유효 샘플이 10경기 미만으로 떨어지는 경우 빈번.
- 분석 결과에 **게임 수(N)** 가 표시될 때 N < 10이면 결과를 참고용으로만 활용.

### 다중공선성
- 시나리오 E의 특성들(15분 골드, 첫 드래곤, 첫 헤럴드, 첫 타워)은 서로 강한 양의 상관관계.
- 하나의 특성이 낮은 계수를 가져도 실제 중요하지 않다는 의미가 아님.

### 시간 의존성 무시
- 선수 이적, 감독 교체, 패치 메타 전환이 모두 데이터에 혼재.
- "시즌 필터"를 적용해도 시즌 내 이적이나 피지컬 변화는 반영되지 않음.
- 최근 경기에 더 높은 가중치를 부여하는 기능 없음.

### 외부 변수 미반영
- 상대 팀의 강도(강팀 vs 약팀 상대 전적 혼재)
- 선수 컨디션, 부상, 교체 출전
- BO5 vs BO1 경기 형식 차이 (BO5는 상대 전략 분석 후 적응 가능)
- 홈/어웨이 (오프라인/온라인) 환경 차이

## 4. 규정 변경 처리 한계

### First Selection (2026년~)
- 선픽 팀이 진영 또는 픽 순서 중 선택하는 신규 규정.
- DB에 `first_selection_choice` 컬럼으로 기록되나, **현재 시나리오 분석에 미반영**.
- 시나리오 E(승리 공식)에서 `is_blue`, `is_first_pick`을 단순 특성으로만 사용하며, 이 둘의 상호작용 분석 없음.

### Fearless Ban-Pick
- 동일 시리즈 내 이미 사용한 챔피언 재픽 불가.
- DB에 `draft_type` 컬럼으로 구분하나, **분석 로직에서 standard/fearless 구분 없음**.
- Fearless 환경에서는 저격 밴(시나리오 D) 전략이 standard와 완전히 다르게 작동하나 동일하게 집계.
- 피어리스 도입 정확한 시점(2025 LCK Cup 이후 추정) 불명확.

## 5. 결과 해석 시 주의사항

분석 결과를 볼 때 아래 체크리스트를 확인하세요.

- [ ] **게임 수(N)가 10 이상인가?** → 미만이면 방향성만 참고, 수치 신뢰 금지
- [ ] **시즌 필터를 적용했는가?** → "전체" 기간 분석은 메타가 다른 시즌이 혼재됨
- [ ] **비교 선수/팀이 비슷한 경기 수를 가지는가?** → 한쪽이 3배 이상 경기가 많으면 단순 비교 불가
- [ ] **G(팀 색깔)의 임계값이 절대적이지 않음을 인지했는가?** → 캐리 의존형/시스템형은 경향성이며, 해당 팀이 매 경기 그 전략을 쓰는 것이 아님
- [ ] **H(스페셜리스트)의 N ≥ 10인가?** → 3~9경기 결과는 우연일 가능성이 높음
- [ ] **E(승리 공식)의 계수를 인과관계로 해석하지 않았는가?** → "오브젝트를 먹으면 이긴다"가 아니라 "이기는 팀이 오브젝트를 먹는 경향"

## 6. 실제로 유용하게 볼 수 있는 데이터

한계가 있더라도 **특정 조건을 만족할 때** 신뢰도 높게 활용할 수 있는 분석입니다.

---

### ✅ 높은 신뢰도 — 사실 기반 집계

#### B. 진영별 챔피언 픽 목록
**왜 믿을 수 있나**: 복잡한 모델 없이 "이 선수가 블루에서 A 챔피언을 X번 픽했다"는 순수 사실만 집계.  
**유용한 상황**:
- 특정 선수의 블루/레드 진영별 챔피언 풀 차이 확인
- 상대 팀 드래프트 준비 시 "레드에서 이 선수가 무엇을 픽할 가능성이 높나" 파악
- 챔피언 풀이 진영에 따라 갈리는 선수 vs 무관한 선수 구분

**활용 조건**: 특정 챔피언의 해당 진영 게임 수가 **5경기 이상**이면 패턴으로 볼 수 있음.

---

#### D. 저격 밴 패턴 — 상대팀별 밴 횟수
**왜 믿을 수 있나**: "A 팀이 B 선수의 챔피언을 X번 early ban에 넣었다"는 단순 카운팅.  
**유용한 상황**:
- 특정 팀을 상대할 때 어떤 선수의 어떤 챔피언이 집중 밴되는지 확인
- 저격 밴 빈도가 높은 챔피언 = 실전에서 상대가 가장 두려워하는 픽
- 밴 전략 수립 시 "우리가 A 챔피언을 봉인하면 상대 선수에게 얼마나 타격인가" 가늠

**활용 조건**: 해당 팀과의 대전 기록이 **8경기 이상**일 때 패턴 신뢰 가능.  
**주의**: 저격 밴율이 높다고 해서 그 챔피언 없이 반드시 진다는 의미는 아님 (→ 시나리오 A로 교차 확인).

---

#### G. 팀 색깔 — 원시 지표값
**왜 믿을 수 있나**: 분류 레이블(캐리 의존형 등)은 임계값이 임의적이나, 그 아래의 수치 자체는 사실.  
**유용하게 볼 수 있는 지표**:

| 지표 | 의미 | 활용 |
|------|------|------|
| **블루 승률 vs 레드 승률 차이** | 진영 의존도 | 차이가 15%p 이상이면 진영 밴픽이 핵심 |
| **선픽 승률 vs 후픽 승률 차이** | 드래프트 주도 능력 | 선픽 승률이 높은 팀 = 자기 픽으로 이기는 팀 |
| **오브젝트율 (드래곤/헤럴드/타워)** | 초반 압박 강도 | 세 오브젝트 모두 50% 이상이면 초반 팀 |
| **두 팀 비교 레이더 차트** | 상대적 강점/약점 | 상성 예측에 가장 직관적 |

**활용 조건**: 해당 시즌 **15경기 이상** 치른 팀 기준.

---

### 🟡 조건부 신뢰도 — 충분한 샘플 전제

#### A. 밴 시 승률 영향
**유용한 상황**: 선수의 주력 챔피언이 실제로 자주 밴될 때.  
**믿을 수 있는 조건**:
- 해당 챔피언이 밴된 경기 수 **10경기 이상**
- 밴 안 된 경기 수도 **10경기 이상**
- 이 두 조건을 모두 만족하면 승률 차이가 5%p 이상일 때 유의미한 패턴

**활용법**: "이 선수의 챔피언을 봉인했을 때 우리 팀 승률이 얼마나 올라가는가"를 계산하는 것이 아니라, "상대 팀의 그 선수를 밴으로 흔들 수 있는가"의 방향성 판단에만 사용.

---

#### F. 밴 내성 — 세부 지표 개별 활용
**총점(0~100)보다 세부 지표를 따로 보는 것이 더 유용**:

| 세부 지표 | 신뢰도 | 의미 |
|---------|-------|------|
| **챔피언 풀 크기** (2경기 이상 챔피언 수) | 높음 | 숫자 자체가 사실. 많을수록 밴 압박에 강함 |
| **주력 챔피언 의존도** (1위 챔피언 비중) | 높음 | 50% 이상이면 그 챔피언 봉인 시 전술 변화 큼 |
| **밴 시 승률 하락폭** | 중간 (N≥10 조건) | 숫자보다 방향성(크다/작다)으로 해석 |
| **골드 변동성 증가** | 중간 | 밴됐을 때 팀 경기 양상이 얼마나 불규칙해지는가 |
| **총 밴 내성 점수** | 낮음 | 가중치가 임의적이므로 팀 내 상대 비교에만 사용 |

**활용법**: 같은 팀 선수들을 나란히 비교해서 "누가 밴 압박에 가장 취약한가" 서열을 파악하는 데 유효. 절대 점수(68점이 좋다/나쁘다)로 해석하면 안 됨.

---

#### H. 스페셜리스트 챔피언 — N ≥ 10 필터링
**총점보다 초과 승률(excess_wr)만 집중**:
- `games ≥ 10` AND `excess_wr ≥ 0.10` (LCK 평균 대비 10%p 이상 높음): **강한 스페셜리스트 근거**
- `games ≥ 10` AND `excess_wr ≥ 0.05`: 참고 가능
- `games < 10`: 방향성만 참고, 수치 무의미

**유용한 상황**: 상대 선수가 특정 챔피언에서 유독 높은 승률을 보일 때, 픽/밴 우선순위 결정에 활용.

---

### 🔴 낮은 신뢰도 — 참고만

#### C. 팀별 패치 적응 속도 순위
- 절대 수치(평균 X경기 만에 채택)보다 **팀 간 상대 순위**만 참고
- 시즌 초반 데이터 부족 시 상위/하위 팀이 뒤바뀔 수 있음

#### E. 승리 공식 계수
- 계수의 **부호(+ / -)** 만 신뢰 (어떤 요소가 승리와 연관되는가 방향성)
- 계수의 **절대값 크기**는 다중공선성 때문에 직접 비교 불가
- "first_herald 계수 > first_dragon 계수이므로 헤럴드가 더 중요하다"는 해석 금지

#### G. 팀 색깔 레이블 (캐리 의존형/시스템형 등)
- 레이블 자체보다 **레이더 차트의 형태**를 보는 것이 더 유용
- "이 팀은 블루 WR은 높지만 레드 WR이 낮다"는 레이더에서 직관적으로 파악 가능

### 📊 가장 실전에 유용한 활용 시나리오

**경기 전 드래프트 준비 시**:
1. **D(저격 밴)** → 상대 팀 어떤 선수의 어떤 챔피언이 자주 밴되는가 확인
2. **B(진영별 성향)** → 우리가 레드/블루를 선택하거나 강요받을 때 상대 선수가 어떤 픽을 할 가능성이 높은가
3. **G(팀 색깔 지표)** → 상대 팀이 블루를 선호하는가, 선픽을 선호하는가 파악

**선수 분석 시**:
1. **F의 세부 지표** → 챔피언 풀 크기 + 주력 의존도로 밴 타겟 우선순위 결정
2. **H(스페셜리스트, N≥10)** → 밴해야 할 선수의 시그니처 챔피언 확인
3. **A(밴 시 승률, N≥10)** → 실제로 그 챔피언을 봉인했을 때 효과가 있는가 검증
